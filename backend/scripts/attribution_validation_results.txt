--- GENERATED ANSWER ---
The resume parsing pipeline in TalentForge is a stateful, twelve-step asynchronous workflow designed to extract high-fidelity structured profiles. It utilizes a "multi-model triangulation" approach, delegating specific tasks to the most suitable tools to minimize hallucinations and ensure data integrity.

### The Pipeline Workflow
1.  **Ingestion & Validation:** Recruiters upload files, which are validated for MIME type using `python-magic`. Files are stored in Supabase, and a task is enqueued to Celery.
2.  **Pre-extraction (Hyperlink Harvesting):** Before OCR, the system uses `fitz` (for PDFs) and `zipfile` (for DOCX) to extract authoritative hyperlink destinations (e.g., GitHub, LinkedIn) directly from document metadata, bypassing the need for LLMs to guess URLs from display text.
3.  **Layout-Aware OCR:** Azure AI Document Intelligence (`prebuilt-layout`) is used to parse text. To prevent "column interleaving" (where multi-column layouts are read incorrectly), the system iterates through `result.paragraphs` in visual reading order rather than raw storage order. Table data is reconstructed row-by-row into a pipe-delimited format.
4.  **Pattern Matching:** A deterministic regex engine runs in parallel to identify emails, phone numbers, and bare-domain portfolios.
5.  **Named Entity Recognition (NER):** The HuggingFace `bert-base-NER` model extracts names, locations, and organizations to provide ground-truth hints for the LLM.
6.  **Synthesis:** The raw text, regex outputs, NER hints, and harvested hyperlinks are injected into `gemini-2.0-flash`, which maps the data to a strict Pydantic schema.
7.  **Deterministic Canonicalization:**
    *   **Experience:** An `experience_calculator.py` module performs date arithmetic, resolves overlapping roles, and calculates tenure using linear month indices.
    *   **Skills:** A tiered `skill_normaliser.py` uses a static alias dictionary for common skills, falling back to a Groq-hosted Llama 3.3 70B model for unknown skills, and finally to a local formatting fallback.
8.  **Embedding & Persistence:** The profile is summarized and converted into a 1536-dimensional semantic vector using OpenAI’s `text-embedding-3-small` model. The final JSON profile and vector are persisted in PostgreSQL.

### Key Design Principles
*   **Separation of Concerns:** The system enforces a strict boundary where LLMs handle unstructured text synthesis, while deterministic Python code handles all business logic (calculations, date math, and skill normalization). This ensures results are reproducible, auditable, and defensible.
*   **Resilience:** The pipeline includes tiered fallbacks. If Azure DI is unavailable, it uses a local parser (PyMuPDF/python-docx) with coordinate-based sorting to maintain reading order. If the Groq API fails during skill normalization, the system degrades gracefully to local formatting.
*   **Data Integrity:** By using deterministic logic for calculations and deduplication, TalentForge ensures that ranking scores are consistent and not subject to the non-determinism of generative models.

QUERY: Explain TalentForge
==========================================
--- ANSWER ---
TalentForge is a smart talent selection engine designed to automate and improve the efficiency of resume screening, a process that is traditionally manual, inconsistent, and time-consuming.

### Core Purpose
The system aims to parse unstructured resumes with high accuracy, rank candidates against job descriptions using a hybrid approach (semantic understanding and hard requirements), and provide transparent, auditable explanations for its ranking decisions.

### Key Architectural Features
*   **Four-Stage Asynchronous Pipeline:** The system processes resumes through a pipeline consisting of:
    1.  **Parsing:** Uses multi-model triangulation (Azure Document Intelligence, HuggingFace BERT, and Regex) to extract data, which is then synthesized by an LLM (Gemini 2.0 Flash) to minimize hallucinations.
    2.  **Canonicalization:** Uses deterministic Python code (not LLMs) for business-critical calculations, such as normalizing skills and calculating precise work experience by merging overlapping intervals.
    3.  **Ranking:** Employs a hybrid engine executed as a single PostgreSQL CTE query. It combines semantic scores (via `pgvector` cosine similarity) with rule-based scores (skills, experience, etc.) at a 40/60 weight split.
    4.  **Justification:** Generates two-sentence, grounded explanations for the top 5 candidates to provide recruiters with immediate insights.

### Design Philosophy
*   **Deterministic vs. Generative:** TalentForge strictly separates concerns: LLMs are used for extraction and synthesis, while deterministic code is used for all business logic and calculations to ensure reproducibility and auditability.
*   **Cloud Economics:** The architecture is designed for "scale-to-zero" cost efficiency. It is built to run on serverless infrastructure (e.g., Azure Container Apps, Neon PostgreSQL, Upstash Redis), keeping projected hosting costs under $25 per month for intermittent, bursty usage patterns.
*   **Data Integrity:** The system treats document metadata as a first-class data source, extracting hidden hyperlinks from binary file structures (PDF/DOCX) rather than relying solely on visual OCR output.
*   **Observability:** The backend utilizes structured JSON logging with unique `request_id`s and distributed tracing to identify latency bottlenecks and ensure operational visibility.

Currently, TalentForge runs locally via Docker Compose, with a target production architecture designed for a fully serverless deployment on Microsoft Azure.

QUERY: Explain ClassSync
==========================================
--- ANSWER ---
**ClassSync** is an automated classwork management platform designed to streamline communication and assignment tracking between teachers and students. It utilizes a decoupled architecture to separate core business logic from background automation and third-party integrations.

### Core Architecture and Tech Stack
*   **Application Layer:** Built with a React 19 frontend and a Node.js/Express 5 backend, using PostgreSQL 16 and Prisma ORM for data management.
*   **Automation Engine:** The system offloads background tasks, cron scheduling, and Telegram messaging to a self-hosted **n8n** instance. This keeps the main API lightweight and responsive by moving intensive tasks (like polling or bulk notifications) out of the primary request cycle.
*   **Infrastructure:** The platform is containerized and managed via Azure Container Apps, utilizing Terraform for infrastructure-as-code and a GitHub Actions CI/CD pipeline for zero-downtime deployments.

### Key Features and Design Principles
*   **Decoupled Services:** ClassSync uses service abstractions for notifications and storage. This allows the system to switch between different providers (e.g., moving from Supabase Storage to AWS S3) with minimal code changes.
*   **Telegram Integration:** The system uses a "Telegram Command Router" in n8n to handle the single-webhook constraint of the Telegram Bot API. It supports features like account linking, assignment notifications, and command-based interactions (e.g., `/assignments`).
*   **Self-Healing Workflows:** To improve efficiency, the system includes a self-healing mechanism for scheduled tasks. If all students have submitted their work, the system automatically updates reminder statuses to prevent redundant database polling.
*   **Data Consistency:** The platform handles complex scenarios, such as students joining mid-course, by using database transactions to backfill missing submission records, ensuring that all past assignments appear on a student's dashboard immediately upon enrollment.
*   **Reliability:** The system employs a "fire-and-forget" pattern for egress channels (like Telegram notifications) to ensure that external API latency does not impact the performance of core user operations, such as file uploads.
